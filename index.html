HTML

  • The head element contains information about the webpage.
  • The information in the head element is called metadata and, aside from the title of the webpage, is not visible to the user.
  • The body element represents the visible content shown to the user.
  • Visible content within the body element includes text, images, hyperlinks, and paragraphing.
  • If a webpage is a house, HTML is the framework that provides structure.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • We can apply CSS properties by referencing HTML elements such as header, footer, and h1; however we can also add styles to custom class selectors and assigning the attributes to certain elements.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add [file]: add a file as it looks now to your next commit (stage).
  • git add -A: Adding all changed files in our working directory to the staging area of Git.
  • git commit -m "insert description": Commit all changes to repository, make sure to add a description for referencing.
  • git restore: unstage a file while retaining the changes in working directory. Used to be called git reset.
  • git pull origin main: Makes sure current branch is up to date before opening a PR.
  • git push origin feature/add-css: Push up all changes in directory. feature/add-css is just an example.
